# Contributing to Orchestrus

## Development Workflow
1. Fork the repository
2. Create a branch for your feature
3. Run `make verify` (includes fmt, lint, test)
4. Submit a PR with RFC link

## Guidelines
- Follow [Zettelkasten] approach for new features
- All changes require performance benchmarks
- Implement 100% test coverage for core components

[Zettelkasten]: docs/rfc-process.md
